The Problem

Every fresh Hermes session is a blank slate

Built-in memory runs out of space. Session search is slow. You repeat yourself, context gets lost, decisions get forgotten. Your Obsidian vault holds your knowledge — but Hermes can't read it.

The Solution

Three pillars of persistent memory

📝

Session Capture

Every user prompt and Hermes response is recorded verbatim in append-only daily session files inside your vault.

🌙

Nightly Summaries

A cron job compresses the day's work into structured summaries — decisions, fixes, errors, and pending tasks.

🔄

Context Recovery

New sessions start informed. The plugin injects the latest summaries so Hermes picks up where you left off.
